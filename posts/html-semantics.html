Understanding HTML Semantic Elements

Published on 18 June 2026 by Patryk Grudziński

When I first started learning web development, my code was a massive jungle of <div> elements. Everything was a div, and honestly, it felt messy. That was until I discovered semantic HTML.

What is Semantic HTML?

Semantic elements are HTML tags that clearly describe their meaning to both the browser and the developer. Instead of using a generic <div id="header">, we use <header>. Instead of a div for navigation, we use <nav>.

Why should you care?

  • SEO (Search Engine Optimization): Search engines like Google use spiders to crawl your website. Semantic tags help them understand what parts of your page are actually important.
  • Accessibility (A11y): Screen readers for visually impaired users rely on these tags to navigate the page properly.
  • Cleaner Code: It is significantly easier to read and maintain code that uses <article>, <section>, and <aside>.
"Clean code always looks like it was written by someone who cares." — Michael Feathers

Moving forward, my goal is to completely banish "div-soup" from my repositories and write clean, semantic code from day one.
